### 📋 模块任务清单：Synthesize with lite agent or LLM

#### 1. 环境准备与触发机制 (Environment & Trigger)
- [ ] **定义回退触发逻辑**：明确定义“执行器路径失效”的具体判定标准（如：超时、工件格式错误、回调证据 `callback evidence` 缺失）。
- [ ] **宿主侧上下文初始化**：确保宿主环境已加载 `https://github.com/666ghj/MiroFish` 的基础元数据和仓库引用（Repository References）。
- [ ] **接入 Lite Agent/LLM 接口**：配置宿主侧轻量级代理（Lite Agent）或直接 LLM 调用链的 API 密钥及连接参数。

#### 2. 元数据处理与分析 (Metadata Processing)
- [ ] **路由元数据提取**：从可用路由（Route Metadata）中解析出核心业务逻辑、目标路径和预期的状态转移。
- [ ] **仓库引用扫描**：分析 MiroFish 仓库中的关键文件结构，提取用于生成蓝图的静态约束和定义。
- [ ] **清理不可信数据**：识别并过滤掉可能由失效执行器生成的断片或伪造的回调数据。

#### 3. 蓝图合成逻辑 (Blueprint Synthesis)
- [ ] **构建“保守蓝图”模版**：设计一种声明式的蓝图格式，重点描述“预期架构”而非“运行状态”。
- [ ] **执行 Lite Agent 启发式合成**：利用轻量级代理对现有元数据进行逻辑推理，填补执行器未完成的逻辑缺口。
- [ ] **LLM 兜底生成**：当 Lite Agent 无法确定的情况下，调用 LLM 根据仓库引用生成建议性的伪代码或逻辑流。
- [ ] **强制完整性校验**：确保生成的蓝图符合 MiroFish 项目的特定规范（Role Agent Path 规范）。

#### 4. 证据合规与真实性控制 (Evidence & Compliance)
- [ ] **实施“非实时”标记**：在输出的蓝图中显式添加元数据标记，声明该工件为“宿主侧合成（Host-side Synthesized）”，而非“实时执行结果（Live Executor Result）”。
- [ ] **证据缺失审计**：检查并剔除任何暗示存在“实时回调证据”的断言或日志。
- [ ] **一致性对齐**：对比原始需求，确保生成的保守蓝图不会超出已知元数据的覆盖范围（不进行无根据的推测）。

#### 5. 输出与交付 (Output & Handoff)
- [ ] **生成工件封装**：将合成的蓝图打包为标准格式（如 JSON 或 YAML），便于后续模块解析。
- [ ] **状态汇报**：记录回退发生的根本原因及合成过程中的信心指数（Confidence Score）。
- [ ] **模块接口联调**：确保输出能够被父模块上下文中的备选路由正确接收。
