# 模块需求文档：敏感数据防护栏 (Apply sensitive-data guardrails)

| 版本 | 状态 | 负责人 | 日期 |
| :--- | :--- | :--- | :--- |
| v1.0 | 草案 | OuYi AI | 2023-10-27 |

## 1. 模块概述
### 1.1 目的
本模块旨在 MiroFish 项目的执行器角色代理（executor-backed role agent）运行过程中，对生成的产出物（Artifacts）和回调摘要（Callback Summaries）进行自动化脱敏处理。其核心目标是在防止泄露真实敏感信息（如 API 密钥、访问令牌、私人凭据、个人身份信息及环境密钥）的同时，保留足够的技术证据以供调试、审计和验证。

### 1.2 上下文说明
- **所属路线**：通过角色代理路径驱动 `https://github.com/666ghj/MiroFish` 并保留实时回调证据。
- **父模块关联**：将已完成的角色代理运行转换为结构化产出物，确保反映主路径并包含足以区分实时运行的证据。

---

## 2. 业务需求
### 2.1 脱敏范围
模块必须能够识别并处理以下类型的敏感数据：
1.  **认证凭据**：API Keys, Bearer Tokens, OAuth Secrets, Password 字段。
2.  **个人隐私 (PII)**：用户真实邮箱、手机号、家庭住址、真实姓名。
3.  **环境敏感信息**：特定服务器 IP 地址、数据库连接字符串、内部配置文件中的私有密钥。
4.  **临时凭证**：Session IDs, CSRF Tokens。

### 2.2 证据保留原则
脱敏过程不得破坏数据的结构和技术参考价值：
- **保留结构**：保持 JSON/XML 的层级结构。
- **保留元数据**：HTTP 状态码（如 200 OK, 403 Forbidden）、响应耗时、API 节点名称。
- **保留逻辑关联**：如果两个不同的地方使用了相同的 Key，脱敏后的占位符应保持一致性（可选），以便于追踪逻辑。

---

## 3. 功能需求

### F-01：自动检测与识别
- 能够基于正则表达式、关键词匹配及上下文分析，自动扫描回调数据中的敏感字段。
- 支持针对特定 API 响应格式（如 JSON、Headers、Log stream）的定制化扫描。

### F-02：智能替换（Scrubbing）
- 将敏感值替换为标准化的占位符，例如：
    - `api_key: "sk-12345..."` -> `api_key: "[REDACTED_API_KEY]"`
    - `email: "user@example.com"` -> `email: "[USER_EMAIL_MASKED]"`
- 替换过程应不可逆。

### F-03：产出物结构化转换
- 在将角色代理运行记录转换为最终 Artifact 时，确保防护栏逻辑作为流水线（Pipeline）的一个强制环节。
- 汇总回调摘要时，仅展示脱敏后的技术路径。

### F-04：证据有效性校验
- 提供一种机制，确保脱敏后的产出物仍包含“实时运行”的证据（如经过哈希处理的唯一标识符或脱敏后的时间戳），以区分模拟运行和真实运行。

---

## 4. 技术要求
- **集成位置**：位于 `executor-backed role agent` 输出层与 `blueprint pipeline` 输入层之间。
- **性能指标**：脱敏处理对整体流水线增加的延迟应小于 500ms。
- **安全性**：脱敏逻辑应在本地/受信任环境执行，不得将原始数据发送至外部第三方脱敏服务。

---

## 5. 验收标准 (Acceptance Criteria)
1.  **安全性验证**：最终生成的 Artifact 中，搜索不到任何真实的 API Key 或 Token 字符串。
2.  **完整性验证**：Artifact 包含完整的 API 调用链路（方法、路径、状态码）。
3.  **可读性验证**：开发人员能够根据脱敏后的回调摘要判断角色代理是否按预期执行。
4.  **符合性验证**：符合 MiroFish 项目对实时回调证据（Live callback evidence）的保留要求。
