以下是该模块的任务清单：

---

### 📋 任务清单：Create execution envelope (构建执行信封)

#### 1. 基础引用与环境准备
- [ ] **解析目标仓库引用**：验证并锁定 `https://github.com/666ghj/MiroFish` 的特定分支、Tag 或 Commit SHA。
- [ ] **提取主路由配置**：从父模块上下文中检索已选定的“主路径”（Primary Route），确保执行逻辑的一致性。
- [ ] **环境引用脱敏处理**：识别载荷中涉及的敏感环境变量，进行脱敏（Redaction）处理，仅保留必要的引用标识而非原始明文。

#### 2. 调用载荷 (Payload) 核心构建
- [ ] **定义请求产物类型 (Artifact Type)**：明确执行后需生成的产物规格（如：日志文件、执行快照、分析报告等）。
- [ ] **设定运行时限制 (Runtime Limits)**：配置执行器的资源约束，包括最大运行时间（Timeout）、内存限制及 CPU 占用配额。
- [ ] **封装调用指令集**：将路由信息、仓库引用和限制条件整合为符合后端执行器接口规范的 JSON/YAML 结构。

#### 3. 回调与证据保留配置
- [ ] **配置回调通道 (Callback Channels)**：建立用于流式传输（Streaming）角色代理活动的实时通道（如 WebSocket 或 Webhook 地址）。
- [ ] **设置证据固化机制**：定义执行过程中的“活跃回调证据”（Live Callback Evidence）保存策略，确保每一步观测到的代理行为都被持久化存储。
- [ ] **初始化状态流监控**：确保执行器在启动瞬间即可向父模块反馈 Initializing 状态。

#### 4. 验证与交付
- [ ] **载荷结构校验**：执行 JSON Schema 验证，确保生成的执行信封（Envelope）符合执行器的输入契约。
- [ ] **模拟链路测试**：（可选）在发送前验证回调通道的可达性。
- [ ] **交付执行信封**：将封装好的载荷传递至后端执行器（Executor-backed role agent path）。

**模块目标总结：**
本模块将生成一个受保护且结构化的配置包，它既定义了 `MiroFish` 项目“跑在哪”、“怎么跑”，又规定了“跑多快”以及“跑完后证据留存到哪”。
