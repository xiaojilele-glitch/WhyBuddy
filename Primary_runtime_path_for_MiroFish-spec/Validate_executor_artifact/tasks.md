### 📋 任务清单：Validate executor artifact (执行器产物验证)

#### 1. 结构与完整性校验 (Structural & Integrity Validation)
- [ ] **定义产物 Schema 标准**：明确不同类型 MiroFish 任务对应的执行器产物标准格式（如 JSON 结构、文件类型等）。
- [ ] **结构合法性检查**：验证产物是否符合预定义的 Schema，确保无语法错误或损坏。
- [ ] **非空检查 (Non-empty Check)**：确认产物包含实质性内容，而非仅返回空对象、空文件或占位符。
- [ ] **关键字段提取**：从产物中提取核心元数据（如 ID、版本、时间戳），用于后续比对。

#### 2. 路由与角色对齐校验 (Route & Role Alignment)
- [ ] **执行路径一致性核验**：对比产物元数据与“Executor-backed role agent”路径的预期输出是否匹配。
- [ ] **意图覆盖率评估**：确认产物是否涵盖了原始任务请求中的所有核心指令。
- [ ] **上下文关联性检查**：验证产物是否基于正确的任务上下文生成，确保没有发生路由偏离（Drift）。

#### 3. 回调证据核查 (Callback Evidence Verification)
- [ ] **收集实时回调数据**：从执行器中获取执行期间保留的 Live Callback Evidence（生存回执证据）。
- [ ] **证据与产物关联分析**：确保证据链能够支撑产物的生成逻辑，验证其“真实性”而非伪造结果。
- [ ] **日志追踪完整性**：检查回调日志中是否存在未捕获的运行时异常（Runtime Exceptions）。

#### 4. 验证决策与路由切换 (Validation Decision & Routing)
- [ ] **定义验证通过准则 (Pass Criteria)**：明确产物被视为“有效”的各项指标阈值。
- [ ] **触发主路径分支 (Primary Route)**：若验证通过，标记任务状态为 `Success`，并传递产物至下一流转节点。
- [ ] **触发回退机制 (Fallback Logic)**：
    - [ ] 识别验证失败的具体原因（结构错误、证据缺失、路由偏离等）。
    - [ ] 自动调用 Fallback 路由（如重试、切换 Agent 或报错提示）。

#### 5. 状态持久化与审计 (Persistence & Auditing)
- [ ] **生成验证报告**：记录各项指标的验证结果（Pass/Fail）及详细的错误描述。
- [ ] **更新 MiroFish 运行状态**：在任务追踪系统中同步当前模块的执行结论。
- [ ] **存档证据链**：将回调证据与最终产物打包存档，以便后续人工审计或模型微调。
